传统处理

传统处理异常多通过try-catch进行捕捉，如此的话会出现code不清晰、异常处理很乱的情况，返回给 移动端、web端接口异常不清晰、不友好等问题

aop

利用aop对所有控制层即controller层 进行代理，形成类似try-catch式异常捕捉处理

spring3.2后提供的注解 @ControllerAdvice @ExceptionHandler注解，实现全局处理异常
